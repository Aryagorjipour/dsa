# Recommended Books

## Core Textbooks

### Introduction to Algorithms (CLRS)
- **Authors**: Thomas H. Cormen, Charles E. Leiserson, Ronald L. Rivest, Clifford Stein
- **Link**: https://mitpress.mit.edu/9780262046305/introduction-to-algorithms/
- **Summary**: The definitive reference. Covers almost every data structure and algorithm in the handbook (Red-Black trees, Segment Trees, Dynamic Programming, Graph algorithms, etc.) with rigorous proofs.
- **Why read it**: Used in top universities. Excellent for deep theoretical understanding. 4th edition (2022) is current.

### The Algorithm Design Manual
- **Author**: Steven S. Skiena
- **Link**: https://www.amazon.com/Algorithm-Design-Manual-Steven-Skiena/dp/1849967202
- **Summary**: Practical focus with "war stories" from real applications. Strong on heuristics, backtracking, and when to use which structure.
- **Why read it**: Bridges theory and practice better than most academic books.

### Algorithms (4th Edition)
- **Authors**: Robert Sedgewick, Kevin Wayne
- **Link**: https://algs4.cs.princeton.edu/
- **Summary**: Excellent for Java/C# style implementations. Very clear explanations of BSTs, Heaps, Graphs, and string algorithms. Comes with superb online course and visualizations.
- **Why read it**: Best for learning clean, production-oriented implementations.

### Grokking Algorithms
- **Author**: Aditya Bhargava
- **Link**: https://www.manning.com/books/grokking-algorithms
- **Summary**: Illustrated, beginner-friendly introduction to the most important algorithms (BFS, Dijkstra, Dynamic Programming, etc.).
- **Why read it**: Perfect warm-up before heavier books. Great diagrams.

### Data Structures and Algorithms Made Easy
- **Author**: Narasimha Karumanchi
- **Link**: https://www.amazon.com/Data-Structures-Algorithms-Made-Easy/dp/819324527X
- **Summary**: Puzzle-oriented with many interview-style problems and solutions.
- **Why read it**: Excellent for interview prep and seeing multiple approaches.

## Advanced / Specialized

- **Advanced Algorithms and Data Structures** (Manning) — Covers probabilistic structures (HyperLogLog, Bloom, Count-Min), succinct structures, etc.
- **Purely Functional Data Structures** (Chris Okasaki) — For immutable versions (relevant to functional languages and persistent data in Git).

**Recommendation order for most readers**:
1. Grokking Algorithms (quick intuition)
2. The Algorithm Design Manual (practical mindset)
3. CLRS (deep reference)
4. Sedgewick (implementations)
